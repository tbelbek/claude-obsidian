## [[do-combination|COMBINATION AB]] — Current CI/CD

Senior Software Engineer, Apr 2025 – Present. I own the CI/CD pipelines and container infrastructure for our .NET 9 microservices. Each service has its own pipeline in **[[ref-github-actions#Reusable Workflows|GitHub Actions]]** with **reusable workflows** from a shared workflow library — build, test, scan, push image, deploy. I restructured all **Dockerfiles** as [[ref-docker#Multi-Stage Builds|multi-stage builds]] — images went from 800MB+ down to 80-100MB. I use **[[ref-kubernetes#Configuration|Kustomize]] [[ref-kubernetes#Configuration|overlays]]** for Kubernetes deployment configs — base with dev/staging/prod overrides — and configure **Kubernetes** deployments with proper [[ref-kubernetes#Health Probes|health probes]] (startup, [[ref-kubernetes#Health Probes|readiness]], [[ref-kubernetes#Health Probes|liveness]]), [[ref-kubernetes#Resource Management|resource limits]] based on actual [[ref-grafana-prometheus#Grafana|Grafana]] metrics, and [[ref-kubernetes#Deployment Strategies|rolling update]] strategies. I manage **Azure Container Registry** with retention policies — before I set up cleanup rules, storage tripled in 3 months. I also maintain the **[[ref-cdeploy#What CDeploy Manages|Python CDeploy framework]]** for deployment orchestration — AKS clusters, CloudFlare DNS, Kubernetes resources, Kafka topics, and GraphQL schema management. Every service is instrumented with [[ref-observability-stack#OpenTelemetry (OTel)|OpenTelemetry]] — traces to [[ref-observability-stack#Grafana Tempo (Distributed Tracing)|Tempo]], logs to [[ref-observability-stack#Grafana Loki (Log Aggregation)|Loki]] via [[ref-observability-stack#Grafana Alloy (Collection Agent)|Grafana Alloy]]. [[ref-testing-strategy#Test Pyramid|Testing]] is standardized with Testcontainers and breaking change detection in CI.

## [[do-volvo|VOLVO CARS]] — Embedded Release Pipelines

Software Factory Engineer, May 2022 – Nov 2023. I owned the release pipeline for **safety-critical embedded automotive software** — the full chain from code push to a tested, gated, [[ref-automotive-compliance#Compliance in Agile|compliance]]-checked release on target hardware. Pipeline stages: build with **Cynosure** (Volvo's internal build framework, with **Cynosure StateDB** for product instance tracking) orchestrated by **ZUUL CI** (Volvo's custom tools — [[do-cynosure-parallel#Sorulursa|if I had to choose mainstream alternatives today]], I'd use GitHub Actions + Artifactory + Kafka), across multiple hardware targets (**QNX ARM**, **Linux x86**, **Android Automotive**), [[ref-cicd-security#Scanning Types|static analysis]], unit tests, **hardware-in-the-loop** integration tests on actual ECU hardware, [[ref-automotive-compliance#ISO 26262 — Functional Safety|regulatory]] gate checks, and staged rollout. Everything auditable.

I maintained the **[[ref-gerrit#Change-Based Workflow|Gerrit]]** instance — project configs, [[ref-gerrit#Access Control|access control]], [[ref-gerrit#Labels & Review|label definitions]], [[ref-gerrit#Labels & Review|submit rules]] — and wrote the **Python trigger daemon** that connected Gerrit events to the build system. I also wrote the **build orchestration layer** in Python that managed parallel execution across targets. The orchestration layer was a Python framework I wrote from scratch — it read a YAML manifest listing all build targets with their resource requirements, set up the correct cross-compilation toolchain for each target (sourcing environment scripts for QNX, Linux, or AAOS), invoked Cynosure's build CLI, parsed build output into structured JSON reports, and managed batch execution — lighter targets in parallel first, then heavier ones that needed more memory for linking. Coordinated between **3+ departments** (regulatory, QA, test) that didn't always talk to each other. Wrote technical docs that met Volvo's strict regulatory requirements. **Mentored teammates** and helped new joiners get up to speed. Cut QA cycle by **30%**. Shipped multiple releases with **zero compliance findings**. Test execution orchestrated with [[ref-ansible#Integration with CI|Ansible]] playbooks against CSP emulators.

## [[do-kocsistem|KOCSISTEM]] — DevOps Transformation From Zero

Software Development Lead → Technology Manager, Mar 2020 – Feb 2022. Full DevOps transformation — **no CI/CD, no code review, no branching strategy** when I started. Built **[[ref-azure-devops#YAML Pipelines|Azure DevOps]]** pipelines with a shared YAML template repository — each project's pipeline was 15-20 lines referencing common stages. Set up **Terraform** for infrastructure as code with [[ref-terraform#State Management|remote state]] on Azure Blob Storage, [[ref-terraform#Plan/Apply Workflow|plan/apply]] separation in the pipeline, and PR comments showing the plan diff.

Integrated **security scanning** into every build — dependency checks, static analysis with tuned [[ref-cicd-security#Quality Gates|quality gates]], [[ref-cicd-security#Secrets Management|secret detection]] with [[ref-cicd-security#Secrets Management|pre-push hooks]]. Migrated all secrets from config files to **Azure Key Vault** and cleaned Git history. Tracked **DORA metrics** ([[ref-dora#Deployment Frequency|deployment frequency]], [[ref-dora#Lead Time for Changes|lead time]], [[ref-dora#Change Failure Rate|change failure rate]], [[ref-dora#Mean Time to Restore (MTTR)|MTTR]]) in **Grafana** dashboards and reported to leadership monthly. Overhauled Git workflow — proper branching, pull requests, code review standards. Went from weekly to daily releases — **7x deployment frequency**. **3 consecutive quarters of zero pen-test findings**. Security team started asking other teams to copy our setup. I also [[ref-code-review#HOW WE SET IT UP|built code review culture from scratch]] with mandatory PRs and quality gates.
