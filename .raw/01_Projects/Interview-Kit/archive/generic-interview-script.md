### EXPERIENCE — The Relevant Parts (3 min)

**.NET Backend at Scale**
> "I've been working with C# backends for a while now — mostly .NET, currently on version 9. I've built REST, GraphQL, and gRPC APIs depending on what fits the job. I took part in moving some systems to the cloud without any downtime and managed to release more often — from weekly to daily. It was a team effort, but I was part of making that happen. I also use Python scripts to automate infrastructure tasks when it makes sense."

**Distributed Systems & Messaging**
> "I've worked with async workflows and RabbitMQ for messaging and background jobs. At Volvo, we had release pipelines where mistakes were expensive, so we had to be careful. I saw what reliability looks like when it really matters. I've also worked with MongoDB and Redis for data storage, depending on the use case."

**Cloud, DevOps & Observability**
> "I try to stay involved in the whole process — not just writing code but also deploying and monitoring it. I use Kubernetes and Docker for containerized workloads, and Terraform for infrastructure-as-code to avoid 'works on my machine' problems. Azure DevOps and GitHub Actions for CI/CD. I set up Prometheus and Grafana at Fiksbilişim so the team could actually see what was happening in production — that visibility matters. Testing and security are just part of the work for me."

**AI/LLM & Code Quality**
> "I use GitHub Copilot daily — after a while you get a feel for when it's helpful and when it's just adding noise. I've also been in teams of around 25 people, doing code reviews and trying to help juniors when they need it. When something's slow, I profile the backend services and fix the root cause rather than patching around it."

---

### LEADERSHIP — Ownership Mindset (2 min)
> "I've been in lead roles with teams up to 25 people, but I still get into the code. I like taking things from idea to production to fixes. When something breaks at 2 AM, I'm usually there figuring it out. In one place, we cut SLA breaches by 60% and doubled release frequency — not by working crazy hours, but by building better systems together. We also got the DevSecOps metrics to 75% implementation and had zero pen test findings across our web apps. At Volvo, I automated the embedded release cycle and cut QA gating time by 30% — in automotive, the tolerance for mistakes is very low, so we had to get that right."

**Key Stats to Drop:**
- Led 25 engineers
- 60% SLA improvement
- 2× release frequency
- Zero pen test findings
- 30% QA cycle reduction (Volvo)
- Registered Scrum Master
