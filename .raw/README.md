# Obsidian Vault

Personal knowledge base with PARA methodology + Knowledge Base structure.

## Structure

```
01_Projects/        → Active projects with deadlines/goals
02_Areas/          → Areas of responsibility (ongoing)
03_Resources/      → Reference materials and notes
04_Archive/        → Completed/inactive items
05_Knowledge_Base/ → Concepts, learnings, evergreen notes
99_Inbox/          → Quick capture (process regularly)
```

## PARA Method

- **Projects**: Short-term efforts with clear outcomes (e.g., "BRF Health Checker v1")
- **Areas**: Long-term responsibilities without deadline (e.g., "Finance", "Career")
- **Resources**: Reference materials (e.g., "Trading Strategies", "Swedish Tax Guide")
- **Archive**: Completed projects, old resources

## Knowledge Base

Evergreen notes, concepts, and learnings that connect ideas across projects.
