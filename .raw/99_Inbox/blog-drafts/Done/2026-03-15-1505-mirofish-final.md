Welcome to MiroFish.

## What MiroFish actually does

MiroFish is a multi-agent simulation system. It creates thousands of AI agents, each with unique traits, biases, and memory systems. These agents interact in simulated environments, debate outcomes, and collectively predict real-world events. The project is built on OASIS, a social simulation engine developed by the CAMEL-AI team [2][3].

The architecture is straightforward but powerful. Individual agents remember past interactions. They form opinions. They influence each other. And when you run enough of them simultaneously, patterns emerge that single models cannot replicate.

## The demo that turned heads

The MiroFish team fed their system the first 80 chapters of *Dream of the Red Chamber*, a classic Chinese novel written in the mid-18th century. The original author died before completing the final 40 chapters. Multiple writers have attempted endings over the centuries. None have achieved consensus.

MiroFish's swarm of agents analyzed character relationships, plot trajectories, and historical context. Then they predicted how the story should conclude. The result? A generated ending that literature scholars are now debating seriously. You can see the full demo at [666ghj.github.io/mirofish-demo](https://666ghj.github.io/mirofish-demo/) [2].

This is not sentiment analysis. This is not summarization. This is a computational model of narrative causality built from simulated social dynamics.

## The technical stack

MiroFish runs on OASIS, which handles agent lifecycle, memory management, and social network effects [3]. The project was created by GitHub user 666ghj and is backed by Shanda Group, a Chinese investment firm with a track record in AI infrastructure [1].

The repository is fully open-source. You can inspect the agent definitions, modify personality parameters, and run your own simulations. The codebase has attracted contributions from developers across prediction markets, academic research, and gaming AI.

## Why this matters for prediction markets

Polymarket and similar platforms rely on human traders aggregating information. MiroFish suggests an alternative: synthetic populations with calibrated biases. If you can simulate how different demographics react to news, you can forecast political outcomes, market movements, or social trends without waiting for real humans to place bets.

The implications are significant. A well-calibrated swarm could process information faster than human markets. It could run 24/7 without fatigue or emotional trading. And it could test counterfactual scenarios that no prediction market can price.

## The open-source advantage

MiroFish's rapid adoption reflects a broader shift. Prediction systems are moving from black-box APIs to inspectable, forkable code. Researchers can verify the methodology. Developers can adapt it for specific domains. Critics can identify failure modes.

The 17,000 stars are not just popularity metrics. They represent a community stress-testing the approach, finding edge cases, and proposing improvements. This is how prediction technology should evolve: in public, with scrutiny, through iteration.

## What comes next

The current demo focuses on narrative prediction. The architecture supports much more. Financial forecasting, policy simulation, epidemiological modeling, and strategic game analysis are all within scope. Each domain requires careful calibration of agent populations and validation against ground truth.

The project is young. Three months is not enough time to prove long-term accuracy. But the methodology is sound, the code is open, and the community is engaged. That combination is rare in AI research.

MiroFish will not replace human judgment. It will augment it with synthetic perspectives we can inspect, debate, and improve. The question is not whether machine swarms can predict the future. The question is whether we are ready to listen to thousands of artificial voices arguing about what happens next.
